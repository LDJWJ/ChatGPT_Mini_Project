ChatGPT는 대화의 맥락을 유지하고 효율적으로 대화를 이어가기 위해 여러 기술을 결합하여 사용합니다. 주요 방법은 다음과 같습니다:

세션 기반 메모리 (Session-based Memory):

ChatGPT는 사용자와의 현재 세션 내에서 대화의 맥락을 유지합니다. 이는 세션 동안 이전의 대화 내용을 기억하고, 이를 기반으로 대화를 이어나갈 수 있도록 합니다.
컨텍스트 윈도잉 (Context Windowing):

ChatGPT는 대화의 특정 부분만을 기억합니다. 이는 대화가 너무 길어질 경우 오래된 메시지를 잘라내고 최근 메시지에 초점을 맞추는 방식으로 이루어집니다. 이를 통해 관련성 높고 일관된 대화를 유지합니다.
순차적 대화 처리 (Sequential Dialogue Processing):

각 사용자의 입력에 대해 순차적으로 응답합니다. 이는 대화가 자연스럽고 일관된 흐름을 유지하도록 합니다.
토큰 제한 (Token Limitation):

ChatGPT는 한 번의 입력과 출력에서 처리할 수 있는 토큰 수에 제한을 두어 효율성을 높입니다. 이는 대화가 너무 길어지는 것을 방지하고 시스템의 부하를 관리하는 데 도움이 됩니다.
대화 내용의 갱신 (Dialogue Content Refreshing):

필요한 경우, 대화의 특정 부분을 갱신하거나 재해석하여 새로운 맥락에 맞게 대화를 조정합니다.
이러한 방법들을 통해 ChatGPT는 사용자와의 대화를 자연스럽고 유용하게 유지하면서, 동시에 시스템 자원의 효율적인 사용을 보장합니다. 이는 사용자에게 일관된 대화 경험을 제공하는 동시에, 시스템의 안정성과 성능을 유지하는 데 중요합니다.


========== 대화 저장
ChatGPT와 같은 대화형 AI 모델은 일반적으로 사용자와의 대화 내용을 임시 메모리에 저장합니다. 이는 세션 중에 대화 맥락을 유지하기 위한 것으로, 세션이 종료되면 이 정보는 일반적으로 사라집니다. 주요 특징은 다음과 같습니다:

임시 메모리 (Temporary Memory):

대화 중에 생성된 데이터는 임시 메모리에 저장됩니다. 이 메모리는 사용자와의 세션 동안 활성화되어 있으며, 세션이 종료되면 데이터는 소멸합니다.
변수 저장:

대화 내용은 변수에 저장되며, 이 변수는 챗봇의 인스턴스 또는 세션과 연결되어 있습니다. 이는 세션 동안 연속성을 유지하기 위해 사용됩니다.
DB 저장이 없음:

ChatGPT와 같은 모델은 사용자의 개인정보 보호와 데이터 보안을 위해 대화 내용을 영구적인 데이터베이스에 저장하지 않습니다. 즉, 대화 내용이 영구적으로 저장되거나 재사용되지 않습니다.
세션 기반 대화 관리:

사용자와의 대화는 세션 기반으로 관리됩니다. 세션은 사용자가 챗봇과 상호작용하는 동안 활성화되며, 세션이 끝나면 대화 내용은 메모리에서 제거됩니다.
개인정보 보호 및 데이터 보안:

사용자의 개인정보 보호와 데이터 보안을 위해, 대화 내용은 암호화되고 일정 시간 후 자동으로 삭제되도록 설계됩니다.
이러한 방식은 사용자의 개인정보 보호를 중요시하며, 또한 시스템의 리소스를 효율적으로 관리하는 데 도움이 됩니다. ChatGPT의 주 목적은 실시간 상호작용과 즉각적인 대화 제공에 있으므로, 장기적인 데이터 저장보다는 현재 진행 중인 대화의 맥락을 유지하는 데 중점을 둡니다.
